Setting up your Pageskin campaign

Ad Trafficking Methods:  We require the use of ad servers as this allows you to manage, filter and control campaigns efficiently.

Ad placement:  Any rich media placement that is able to deliver a JavaScript creative can be used to serve the Pageskin Tag. For many publishers this is likely to be a 1x1 or out-of-page placement.


As our skin does not sit in the iframe we are served into, the slot will leave a whitespace, please let us know if you required us to collapse this whitespace when our skin serves.

Layout Support:  This integration only works with the layout of your homepage. Let us know if there are other layouts you would like us to support.

Testing the integration

Serve the campaign to a test page or a live page with cookie or IP targeting (194.74.140.165). This is important so that the test campaign is not delivered to your users.


DFP specific:  Please do not use the option the DFP preview a creative for creating a test page as it does not support targeting and part of the testing is to check that the targeting settings have been set up correctly. This is important as we can only monetise an impression when it meets our targeting criteria.


Once this is complete email your Integration Engineer a link to the page where you are serving the Pageskin campaign.

Important note: This test tag should NOT be served to your live site. You will be sent a new ad-tag for live campaigns.
